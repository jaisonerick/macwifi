# Code of Conduct

This project aims to be useful, direct, and respectful. Everyone participating
in issues, pull requests, discussions, and reviews is expected to help keep the
project constructive.

## Expected Behavior

- Be respectful and patient.
- Assume good intent, but clarify when something is ambiguous.
- Keep feedback specific, technical, and actionable.
- Respect user privacy and do not ask people to share sensitive WiFi or
  Keychain data publicly.
- Accept maintainer decisions about project scope, security posture, and
  macOS privacy boundaries.

## Unacceptable Behavior

- Harassment, insults, threats, or discriminatory language.
- Publishing another person's private information.
- Pushing for ways to bypass macOS consent, Keychain prompts, or other privacy
  controls.
- Repeatedly derailing issues or pull requests after maintainers ask to stop.
- Spam or low-effort promotion unrelated to the project.

## Reporting

If you see behavior that violates this code of conduct, contact the maintainer
privately when possible. If no private channel is available, open a public issue
that asks for a private contact path without including sensitive details.

Maintainers may remove comments, close issues, reject contributions, or block
participants when needed to protect the project and its users.
